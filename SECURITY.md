# Security Policy

The Peel team takes security seriously. We appreciate your efforts to responsibly disclose any vulnerabilities you find.

## Supported Versions

Only the latest `master` branch is currently supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| master  | :white_check_mark: |
| < 0.1.x | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability in Peel, please send an email to **security@peel-lang.org**. 

### What to include

-   A description of the vulnerability.
-   Steps to reproduce the issue (a proof-of-concept script is highly appreciated).
-   The potential impact of the vulnerability.

### What to expect

-   You will receive an acknowledgment of your report within 48 hours.
-   We will work with you to understand and address the issue.
-   We will notify you once the vulnerability is fixed.

We ask that you do not disclose the vulnerability publicly until we have had a chance to remediate it.

Thank you for helping keep Peel secure!
